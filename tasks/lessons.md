# lessons.md — 朝ニュースPodcastプロジェクト

## 2026-07-07
- **音声スタイルの確定**: 当初「2人掛け合い」を提案したが、ユーザーの最終希望は
  「1人ナレーターがカジュアルに語る」形式。→ 掛け合い前提の設計を捨て、単一話者で再設計。
  教訓: 試作を見せてから確定させる方針は正解だった。掛け合いは作り込みやすいが、
  ユーザーによっては「1人の落ち着いた語り」を好む。決め打ちしない。
- **尺**: 約10分・ニュース12本。単一話者で10分はTTSの結合尺管理が重要
  （1本あたり40〜50秒目安で台本の文字数を制御）。
- **配信時刻**: 毎朝6時（JST）。GitHub ActionsのcronはUTC指定なので前日21:00 UTC。
  Actionsは混雑時に数十分遅れる点を手順書に明記すること。
- **ユーザーは非エンジニア**: 専門用語には必ず日本語の補足。PC操作は画面単位で丁寧に。

## 2026-07-08（本番デプロイ時の学び）
- **尺の指示は文字数で**: 「1本40〜50秒」「全体約10分」という時間ベースの指示では
  Geminiが2,253文字（約5分）しか書かなかった。「1本300〜360文字・全体3,800〜4,500文字」と
  文字数で明示＋短い場合の自動増量リトライを入れたら3,941文字（747秒）になった。
  教訓: LLMへの長さ指定は時間でなく文字数で。保険のリトライも入れる。
- **フォールバック実証**: 同日2回実行したらGemini TTSの無料枠に達し、2回目は設計どおり
  edge-ttsに自動切替して完走。両経路とも本番で動作確認済み。
- **デプロイはPAT＋API全自動が最速**: 非エンジニアのユーザーでも、トークン発行だけ
  お願いすれば repo作成→push→Secret登録→Pages有効化→dispatch→検証まで完全代行できる。

## 2026-07-08（番組リニューアル）
- ジャンルを「政治(NHK)・経済(Yahoo)・テック(Yahoo)」各4本に変更。
- トーンを「アナウンサー・事実報道・感想なし」に変更。感想の禁止は
  具体例（「〜だと嬉しいですね」等）を挙げて指示すると効きが良い。
- NHKのRSSは301転送されるがfeedparserが自動追跡するので問題なし。
- 検証方法: push前にローカルでfetch+台本生成を実走し、NG語（ですね/嬉しい等）の
  簡易チェックで感想混入を機械的に確認した。
