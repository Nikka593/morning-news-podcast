朝ニュースPodcast セットアップ手順書

プログラミング不要。画面の指示どおりに進めれば、毎朝6時に自動で番組が作られ、スマホに届くようになります。所要時間の目安：30〜40分（初回のみ）

この仕組みの全体像
① GitHub（ギットハブ：無料のプログラム置き場）に、お渡しするファイル一式を置く
② GitHubが毎朝6時に自動で「ニュース収集 → 台本作成 → 音声化 → 配信」を実行
③ できた音声はネット上のURLに公開され、スマホのPodcastアプリに毎朝届く
※すべて無料枠の範囲。パソコンは初回設定のときだけ使い、あとは電源を切っていてもGitHub側で動きます。
1

GitHubアカウントを作る（持っていれば飛ばしてOK）

  1. https://github.com を開く
  2. 「Sign up（サインアップ）」からメールアドレス・パスワードを登録
  3. 登録したユーザー名（例：naotaro）を覚えておく。後でURLに使います。
2

お渡ししたファイルを置く「リポジトリ」を作る

リポジトリ＝プロジェクト1つ分のファイル置き場のこと。

  1. GitHub右上の「＋」→「New repository（新しいリポジトリ）」
  2. Repository name（名前）に morning-news-podcast と入力
  3. 「Public（公開）」を選ぶ
    ※Podcastアプリから読むにはこの設定が必要です。URLを知っている人だけがアクセスできる状態で、検索には出ません。
  4. 「Create repository」を押す
3

ファイル一式をアップロードする

  1. お渡しした morning-news-podcast.zip をパソコンで解凍する
  2. 作ったリポジトリの画面で「uploading an existing file（既存ファイルをアップロード）」のリンクをクリック
  3. 解凍したフォルダの中身をすべて、画面にドラッグ＆ドロップ
    ※フォルダごとではなく、中のファイル・フォルダを選んでまとめて入れます。
  4. 下の「Commit changes（変更を確定）」を押す
4

あなたのユーザー名をURLに反映する

config.yaml という設定ファイルの中の Nikka593 を、あなたのユーザー名に書き換えます。

  1. リポジトリ内の config.yaml をクリック → 右上の鉛筆マーク（Edit）
  2. base_url: の行の Nikka593 を自分の名前に変更
    例：https://naotaro.github.io/morning-news-podcast
  3. 同じく docs/index.html と docs/setup_guide.html 内の Nikka593 も同様に置き換え（任意）
  4. 「Commit changes」で保存
5

Gemini（無料AI）のAPIキーを取得して登録する

APIキー＝AIを使うための「合言葉」。台本と音声の品質を上げるために使います。無くても動きますが、あると自然になります。

  1. https://aistudio.google.com/apikey を開き、Googleでログイン
  2. 「Create API key（APIキーを作成）」を押し、表示された文字列をコピー
    この文字列は他人に見せないこと。
  3. リポジトリ画面上部の「Settings（設定）」タブへ
  4. 左メニュー「Secrets and variables」→「Actions」を開く
  5. 「New repository secret」を押す
  6. Name に GEMINI_API_KEY、Secret に コピーしたキーを貼り付け →「Add secret」
6

自動実行（GitHub Actions）を有効にする

  1. リポジトリ上部の「Actions」タブを開く
  2. 「I understand my workflows, go ahead and enable them（有効化）」が出たらクリック
  3. 左の「朝ニュースPodcastを自動生成」を選ぶ
  4. 右の「Run workflow」ボタンで、今すぐ手動でテスト実行できます
  5. 数分後、緑のチェックが付けば成功。赤い×が出たらログを確認（多くはAPIキーの貼り間違い）。
7

公開ページ（GitHub Pages）を有効にする

  1. 「Settings」→ 左メニュー「Pages」を開く
  2. Source を「Deploy from a branch」に
  3. Branch を main、フォルダを /docs に設定 →「Save」
  4. 1〜2分待つと、公開URLが表示されます
8

スマホのPodcastアプリに登録する

あなたの番組URL（末尾が /feed.xml）を控えます。
例：https://naotaro.github.io/morning-news-podcast/feed.xml

iPhone（Apple Podcast）の場合：

  1. Podcastアプリ →「ライブラリ」タブ
  2. 右上「・・・」→「番組をURLで追加」
  3. 上のURLを貼り付けて「追加」

これで、毎朝6時すぎに新しい回が自動で届くようになります。

覚えておくと便利なこと
・ニュースの種類を変えたい → config.yaml の feeds: を編集
・語り口を変えたい → config.yaml の tone: を編集
・男性の声にしたい → gender: "male" に変更
・配信が6時ぴったりでない → GitHubの自動実行は混雑時に数十分遅れることがあります（仕様です）
・編集はすべてブラウザ上でOK。パソコンにソフトを入れる必要はありません。
